ARCHITECTURE OVERVIEW (UPDATED - NOW INCLUDES BACKEND)
================================================================================

The reactive architecture using BehaviorSubject and async pipes has been
maintained throughout all changes. This is the core pattern:

FRONTEND ARCHITECTURE (Client-side):
    EmployeeService (HTTP + Reactive)
    └─ BehaviorSubject: employees$
       └─ Observable<Employee[]> stream
          └─ EmployeeList Component
             └─ employees$: Observable consumed by async pipe
                └─ Template: *ngIf="employees$ | async as emp"
                   └─ *ngFor="let emp of (emp | employeeSearch:searchTerm)"

BACKEND ARCHITECTURE (Server-side):
    Express Server (http://localhost:3000)
    └─ SQLite Database (emp_database.db)
       └─ Employees Table (id, name, department, salary, email, timestamps)
       └─ API Routes:
          ├─ GET /employees → SELECT * FROM employees
          ├─ GET /employees/:id → SELECT * FROM employees WHERE id = ?
          ├─ POST /employees → INSERT INTO employees
          ├─ PUT /employees/:id → UPDATE employees WHERE id = ?
          └─ DELETE /employees/:id → DELETE FROM employees WHERE id = ?

DATA FLOW (End-to-End):
    Angular Component
       ↓ (Request via HTTP)
    EmployeeService
       ↓ (HTTP GET/POST/PUT/DELETE)
    Express Server
       ↓ (SQL Query)
    SQLite Database
       ↓ (Query Result)
    Express Server
       ↓ (JSON Response)
    EmployeeService (tap → BehaviorSubject)
       ↓ (Observable Stream)
    Angular Component (async pipe subscription)
       ↓ (Automatic UI Update)
    User sees updated data

REACTIVE PATTERN MAINTAINED:
    ✓ Single source of truth: BehaviorSubject
    ✓ Async pipe prevents memory leaks
    ✓ Unidirectional data flow
    ✓ Automatic UI updates on data changes
    ✓ Real backend persistence with SQLite
    ✓ HTTP error handling with fallbacks

DOCKER ORCHESTRATION:
    docker-compose up
         ↓
    Starts two services on shared network
         ↓
    Backend (port 3000) + Frontend (port 80)
         ↓
    Nginx proxies API requests (/api/* → http://backend:3000/*)
         ↓
    Database persists in volume
         ↓
    Full stack accessible at http://localhost
