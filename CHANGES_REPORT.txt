ARCHITECTURE OVERVIEW (PRESERVED)
================================================================================

The reactive architecture using BehaviorSubject and async pipes has been
maintained throughout all changes. This is the core pattern:

    EmployeeService
    └─ BehaviorSubject: employees$
       └─ Observable<Employee[]> stream
          └─ EmployeeList Component
             └─ employees$: Observable consumed by async pipe
                └─ Template: *ngIf="employees$ | async as emp"
                   └─ *ngFor="let emp of (emp | filter)"
                      └─ Custom Pipe (departmentFilter)
                      └─ Custom Directive (highlightSalary)

Dialog flow:
    EmployeeList Component
       └─ viewEmployee(emp) calls dialog.open(EmployeeDetail, { data: emp })
          └─ MatDialog Service
             └─ CDK Overlay Container (backdrop overlay applied)
                └─ EmployeeDetail Dialog Component
                   └─ Receives employee via @Inject(MAT_DIALOG_DATA)
                   └─ onClose() calls dialogRef.close() → dialog closes

Data flow remains unidirectional and reactive throughout.
